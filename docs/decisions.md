## D1 — 检测模式

**决策：先仅 `person`**

- 阶段一 `core/` 与推理后处理只处理 person 类（COCO class 0）。
- `object` / `anomaly` 模式暂不实现，后续按需扩展。

---

## D2 — 相机与视频源

**决策：USB 相机 + 支持导入录制视频进行检测**

| 来源 | 阶段 | 说明 |
|------|------|------|
| USB (V4L2/GStreamer) | 阶段二 | 现场主链路，`camera/v4l2_usb.py` |
| 录制视频文件 | 阶段一末 / 阶段二初 | 调试、回放、离线验证；`camera/video_file.py` |

- 统一抽象 `CameraStream`，USB 与视频文件共用同一检测流水线。
- 视频文件用途：无相机时的开发联调、难 case 回放、阈值整定；**不替代**现场 USB 实时采集。
- Orbbec 深度相机：本期不做。

---

## D3 — PLC

**决策：西门子 S7-1200 / S7-1500**

- 通信库：`python-snap7` ≥ 3.0。
- 协议模式（block / command）待与 PLC 程序对接时二选一或并存。
- **待现场确认**：IP、rack、slot、DB 号、PUT/GET 是否开启（关闭时需 S7CommPlus 路径）。

---

## D4 — 录像

**决策：快照 + 短片段（默认）**

- 报警边沿（SLOW/STOP）触发快照 + 预录缓冲内短片段。
- 不做长时间连续录像（Orin Nano 无 NVENC）。

---

## D8 — 训练算力

**决策：A100 不在厂内；调试人员笔记本有 GPU → 现场闭环用本机训练**

| 角色 | 机器 | 训练 / 数据 |
|------|------|-------------|
| 研发 | A100（SSH，厂外） | 仅算法开发、脱敏样例、ONNX 基线；**现场生产数据不出厂** |
| 调试人员 | Windows 笔记本 GPU | `windows_studio` 复核 + **LocalCudaBackend 本机微调** + ONNX 下发 Jetson |
| 现场 | Jetson | 收 ONNX → 编 engine → 冻结测试集召回验收 |

- 阶段三 `TrainBackend` 默认：`local`（Win 本机 CUDA）。
- 不实现厂外 A100 承接现场难 case 训练（数据合规）。
