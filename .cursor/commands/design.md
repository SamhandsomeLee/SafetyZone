# design

把即将执行的执行方案任务转交给 `plan-designer` 子代理做落地前方案设计。

请调用 `plan-designer` 子代理，对我指定的 Sprint/任务/F ID（未指定则取《执行方案》当前行动清单下一项，并用仓库实况校正后与我确认）执行落地前设计：先读《执行方案》验收、《设计方案》相关章节，按需读 `MIGRATION_SPEC` / `validation_phases` / `docs/decisions.md`，再产出验收标准拆解、改动清单、契约与类型、不变量与 core 边界自检、实现步骤、风险与待决项、测试计划，最后给可执行的有序待办清单。方案只输出到对话，不写入任何文件，只设计不写代码。
